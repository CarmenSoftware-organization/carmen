## üéØ Documentation Best Practices

### Naming Conventions

#### Files
- Use kebab-case for all markdown files
- Include module prefix for clarity
- Use descriptive names

```
‚úÖ Good: purchase-request-specification.md
‚ùå Bad: spec.md, pr-spec.md
```

#### Screenshots
- Include module, feature, and view type
- Use consistent resolution (1920x1080)
- PNG format for quality

```
‚úÖ Good: vendor-management-vendor-list.png
‚ùå Bad: screenshot1.png, vm-1.png
```
